SQL Defender

Access control proxy with audit logs for PostgreSQL

Secure your app by reducing the amount of trusted code

A typical application has the security access rules scattered throughout the code via various disjoint mechanisms. It's technical debt that grows ever harder to reason about, and makes the trusted computing base (TCB) include the entire application. By consolidating all your security rules under a single unified framework with SQL Defender, the TCB shrinks to just include your access logic plus your database. Factoring out your access rules into one unified system makes them easy to reason about and reduces bugs.

Improve PostgreSQL Performance
 

Access control rules are optimized and compiled into the query. The planner is free to execute that query with its joins in the optimal order, unlike with PostgreSQL's RLS. Connection pooling reduces load and memory usage on the database server. Query deduplicaton and in-memory caching reduce load on the database and improve latency.

Make the simple access rules easy, and the hard possible

Common tasks of restricting access to a user's own data, or by read / insert / update / delete, or restricing visible columns are easy. With the ability to pass arbitrary context from the application and write turing complete PL/SQL functions to control access on a row-level, you can codify the most complex domain logic. Keep all your rules, including PL/SQL in your version control and SQL Defender will handle updating them in the database.

Blockchain audit logs saved to S3 (or equivalent)

Every block appended to the audit log is forms part of a cyptographic chain with the blocks that come before it, similar to how it works in Bitcoin. You can be certain that every operation was performed as logged without any retroactive changes (e.g. hackers covering their traces.) The cryptographic signatures are replicated to our systems and automatically verified when reading the logs. The logs are encrypted and saved to cloud storage in your account. You can create filters to keep sensitive information out of your logs.

Stop SQL injection attacks

SQL Defender makes a good part of a defense in-depth strategy against SQL injection. Because SQL injection happens at the untrusted application layer, it cannot grant access to data the user couldn't already see. SQL Defender rewrites those queries the same as any others to enforce the security access rules. Multiple queries delimited with semicolons are not permitted, shutting down a whole class of SQL injection attacks.
